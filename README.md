# 🧩 Multi-Agent Investment Research System (Graph RAG)

---

## 🚀 Built using LLM-based agents, Neo4j, and Graph RAG principles

---

## 🚀 Overview
A hierarchical multi-agent AI system designed to answer complex investment and market intelligence queries using Neo4j-based Graph RAG and specialized reasoning agents.

---

## 🧠 Problem
Traditional RAG systems struggle with:
- Structured financial data  
- Relationship-heavy queries  
- Multi-step reasoning  

This system addresses these challenges using **specialized agents** combined with **graph-based retrieval**.

---

## 🏗 Architecture
The system uses a hierarchical multi-agent architecture where a central **Root Agent** orchestrates query routing and response aggregation. 

Specialized agents handle different responsibilities:
- A **Graph Database Agent** performs Neo4j-based graph retrieval for complex relational queries  
- An **Investor Research Agent** manages investment relationships and portfolio analysis  
- An **Investment Research Agent** uses pre-defined tools for domain-specific insights  

The system maintains conversational context and aggregates outputs into structured, accurate responses.

---

## 🤖 Agents Explained

### 🧭 Root Agent (Orchestrator)
- Routes queries intelligently  
- Maintains conversational context  
- Aggregates responses from multiple agents  

### 🧩 Graph Database Agent
- Converts natural language → Cypher queries  
- Handles multi-hop graph traversal  
- Performs complex analytical queries  

### 📊 Investor Research Agent
- Tracks investment relationships  
- Retrieves investor portfolios  
- Analyzes investment patterns  

### 📈 Investment Research Agent
- Uses pre-defined MCP tools  
- Fetches industries, companies, and sentiment data  
- Optimized for common research workflows
